Microsoft Entra Agent ID introduced two identity types: [agent identities](../../agent-id/agent-identities.md) and [agent identity blueprint principals](../../agent-id/agent-blueprint.md). These identity objects derive from service principals, and so carry the same requirements and best practices for ownership, lifecycle management, and cleanup as any service principal. Having a designated owner for these objects helps in two important areas of agent identity management: 

- Risks are contained and investigated by a responsible party
- Disabled objects don't introduce dormant privileges

The owners relationship on each agent identity object designates the human users responsible for technical operations and incident response. So when ID Protection flags an agent identity as risk or anomalous resource access patterns emerge, the security operations team can route to a responsible party for containment and investigation. If there's no owner designated, a compromised agent can remain active and continue to pose risks until the next manual directory audit, giving threat actors extended dwell time.

When an agent identity or agent identity blueprint is disabled, it can't acquire new tokens so it can't access resources. The object still exists in the directory, so its app role assignments, group memberships, and OAuth2 permission grants persist. If the agent identity or agent identity blueprint is reenabled by administrative error or a threat actor with directory write access, every permission snaps back without re-approval.

The second failure mode is disabled objects that were blocked as part of triage but never deleted: a disabled object's app role assignments, group memberships, and OAuth2 permission grants persist in the directory. If re-enabled by administrative error or a threat actor with directory write access, every permission snaps back without re-approval. The combination of ownerless objects and stale disabled objects creates a standing-privilege accumulation pattern that proper ownership and cleanup are designed to prevent.

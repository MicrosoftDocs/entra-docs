When an organization deploys AI agents, those agents acquire access tokens for organizational resources on every interaction without an interactive user session and without the device, location, or MFA signals that classic Conditional Access uses to make trust decisions for human users. Microsoft Entra Agent ID introduces two distinct identity types: agent identities (instantiated agents modelled as service principals that perform agentic tasks) and agent users (non-human user accounts that back agent experiences requiring a mailbox or Teams presence). Conditional Access treats them as separate principal types — a policy targeting agent identities does not evaluate on agent-user sign-ins and the reverse is also true. A tenant that enables agent workloads without at least one Conditional Access policy enforcing block-unless-approved has no enforcement boundary on autonomous AI access: every token request from an agent identity or agent user is allowed by default, exactly the failure mode adversaries exploit when they compromise a single agent and pivot through the resources it can reach.
